# Gitmoji コミットメッセージ作成

## 禁止事項

`git add`、`git rm`、`git restore` 等のステージング操作は一切実行しない。未ステージの変更やUntracked ファイルがあっても無視し、ステージ済みの内容だけを対象にする。

## ワークフロー

1. `git diff --cached` でステージ済み変更を確認する
2. 決定木に従って gitmoji を選択する
3. コミットメッセージを作成し `git commit` を実行する

## ユーザーに確認するケース

以下の場合は自分で判断せず、AskUserQuestion ツールでユーザーに確認する:

- **絵文字の選択に迷うとき** — 複数の gitmoji が当てはまりそうな場合
- **コミットを分割すべきとき** — 性質の異なる変更が混在している場合

## コミットメッセージのフォーマット

```
<emoji> <日本語での簡潔な説明>
```

1行のみ。詳細や本文は書かない。

## 決定木（優先順位順に評価）

以下の順序で判定し、最初に一致したものを使用する:

1. **🚚 ファイル移動・リネーム** — `git diff --staged --name-status` が `R`、コード内容の変更なし
2. **🤖 エージェント・プロンプト更新** — `.claude/`、`CLAUDE.md`、スキル定義の変更
3. **☁️ IaC 変更** — `*.tf`、`terraform/` 配下のインフラ定義ファイルの変更
4. **📝 ドキュメント更新** — `*.md`、`docs/`、コメントのみの変更（プログラム動作に影響しない）
5. **🔧 設定ファイル更新** — `*.json`、`*.yaml`、`*.toml`、CI/CD 設定（ロックファイルを除く）
6. **📦 依存関係の更新** — `*lock*`、`requirements.txt`、外部ライブラリの変更
7. **✅ テストの更新** — `test_*.py`、`*.test.ts`、`tests/*`（本番コードの変更なし）
8. **✨ 新機能の実装** — 完全に新しい機能、新しい API エンドポイント
9. **♻️ リファクタリング** — 外部動作を変えずにコード品質を改善
10. **🐛 バグ修正** — 不正な動作を修正
